Goal: develop and deploy a first release as soon as possible

Requirements:
- Keep in mind educators are not developers, the first release should not incorporate too many git functions, but just keep the must ones. In the future, more git features can be added but still need to balance learning curve and user experience v.s. functionalities. Essentially, the OER ecosystem does not need to be as strict as software engineering. usefulness (in the context of OER development and sharing) is the top priority. 
- Modularized design, only serve core functions/features to make the product usable demonstrating the concepts, more functions/features will be added later.
