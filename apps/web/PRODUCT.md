# Product

## Register

product

## Users

STEM educators — chemistry first — who are **not developers**. They arrive
between classes, on office desktops, laptops, and sometimes phones, to write
and organize course materials. Their job to be done: turn messy teaching
materials into structured, publishable open educational resources without
learning Git, Markdown tooling, or web publishing. Secondary viewers: students
and colleagues who open published pages and the public discovery index.

## Product Purpose

Alembic is an OER authoring and publishing platform: educators organize
knowledge and pedagogy; the platform handles structure, versioning, GitHub
publication, provenance, and reuse. The name is the chemist's distillation
vessel — raw materials in, refined reusable resources out. Success: a
non-developer chemist finds the create → edit → publish loop natural and
trustworthy.

## Brand Personality

Scholarly, calm, craftsmanlike. "A teaching-material studio, not a code
editor." The interface should feel like a well-organized workbench in a
library: quiet surfaces, one warm copper note (the alembic vessel), serif
display type over a working sans. Emotional goals: trust, competence, and a
sense that publishing is a considered, deliberate act.

## Anti-references

- Developer tools (GitHub's own UI, VS Code): never surface Git/repo concepts
  in primary flows; the vocabulary is save, preview, publish, snapshot, adapt.
- Generic AI-SaaS gloss: purple-blue gradients, glassmorphism, hero metrics,
  card-grid landing pages.
- LMS dashboards (Canvas/Moodle density and chrome) — Alembic is an authoring
  studio, not an admin console.

## Design Principles

1. **The tool disappears into the writing.** Editors get maximum quiet space;
   chrome collapses, never competes with content.
2. **One warm note on a cool ground.** Copper is the only decorative hue; it
   marks actions, selection, and focus — everything else is tinted neutral.
3. **Publishing is ceremonial.** Tier-3 actions (publish, list publicly) look
   deliberate and confirmable, never one-tap casual.
4. **Educator vocabulary everywhere.** Labels, errors, and empty states speak
   teaching, not tooling.
5. **Works on the podium phone.** Every screen must at minimum read and
   navigate well at 375 px; editing is optimized for ≥768 px but never breaks
   below it.

## Accessibility & Inclusion

WCAG 2.1 AA: body text ≥4.5:1 in both themes (verified numerically for the
token palette), visible focus rings, reduced-motion support, alt text
required on assets (the platform itself asks for it), no color-only state
signals. Touch targets grow under `pointer: coarse`.
